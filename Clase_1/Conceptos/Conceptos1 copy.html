1. Fundamentos Web y Full Stack
¿Qué es Full Stack?
Perfil que comprende varias capas de una aplicación: frontend, backend, base de datos y herramientas asociadas. No implica ser el mayor especialista en cada capa, sino entender cómo se integran.
¿Qué es un stack tecnológico?
Es el conjunto de tecnologías que se usan para construir una aplicación. Por ejemplo: HTML, CSS, JavaScript, Node.js y una base de datos.
¿Qué es el frontend?
Es la capa visual con la que interactúa el usuario. Presenta la información y captura eventos como clics, escritura o navegación.
¿Qué es el backend?
Es la capa que implementa la lógica de negocio, valida reglas, procesa pedidos del cliente, accede a datos y devuelve respuestas.
¿Qué es la base de datos?
Es el sistema donde la aplicación persiste información de forma organizada, por ejemplo usuarios, productos, ventas o mensajes.
¿Qué es el cliente?
Es el entorno desde donde el usuario accede a la aplicación, normalmente un navegador web o una app.
¿Qué es HTTP?
Es el protocolo de comunicación que utilizan cliente y servidor para intercambiar información en la web.
¿Qué es un request?
Es una solicitud que el cliente le envía al servidor para pedir o enviar información.
¿Qué es un response?
Es la respuesta que el servidor devuelve al cliente después de procesar un request.
¿Qué es JSON?
Es un formato liviano de intercambio de datos basado en pares clave-valor. Se usa mucho entre frontend y backend.
Diferencia entre sitio web y aplicación web
Un sitio web suele enfocarse en mostrar información. Una aplicación web es más interactiva y permite que el usuario realice acciones y trabaje con contenido dinámico.
¿Qué es una web estática?
Es una web que muestra esencialmente el mismo contenido para todos los usuarios y no depende de lógica dinámica del lado del servidor.
2. HTML
¿Qué es HTML?
Es un lenguaje de marcado que define la estructura y el significado del contenido de una página web.
¿HTML es un lenguaje de programación?
No. HTML no programa lógica; estructura contenido mediante etiquetas.
¿Qué es una etiqueta HTML?
Es una marca que define un elemento del documento, por ejemplo <p>, <h1> o <form>.
¿Qué es un atributo HTML?
Es información adicional que se escribe en la etiqueta de apertura de un elemento, por ejemplo id, class, src, href o type.
¿Qué hace <!DOCTYPE html>?
Declara que el documento usa HTML5 y le indica al navegador cómo interpretar correctamente la página.
¿Qué es <html>?
Es el elemento raíz del documento HTML. Contiene todo el contenido de la página.
¿Qué es <head>?
Contiene configuración no visible para el usuario, como título, metadatos, enlaces a CSS y scripts.
¿Qué es <body>?
Contiene el contenido visible de la página: textos, imágenes, listas, formularios, botones, etc.
¿Qué es <meta>?
Es una etiqueta que define metadatos del documento, es decir, información sobre la página que no se muestra directamente al usuario.
¿Qué hace <meta charset="UTF-8">?
Define la codificación de caracteres del documento, permitiendo mostrar correctamente tildes, ñ y símbolos.
¿Qué hace <meta name="viewport">?
Ayuda a que la página se adapte correctamente al ancho del dispositivo, especialmente en celulares y tablets.
¿Qué hace <title>?
Define el texto que aparece en la pestaña del navegador.
¿Qué hacen <h1> a <h6>?
Son encabezados jerárquicos. h1 representa el nivel más importante y h6 el menor.
¿Qué es <p>?
Representa un párrafo de texto.
¿Qué hace <br>?
Inserta un salto de línea dentro del contenido.
¿Qué hace <hr>?
Inserta una separación temática, generalmente mostrada como una línea horizontal.
¿Qué es <a>?
Es la etiqueta de enlace. Usa href para indicar el destino.
¿Qué es <img>?
Inserta una imagen. Usa src para la ruta y alt para texto alternativo.
¿Qué es <ul>?
Es una lista desordenada, normalmente mostrada con viñetas.
¿Qué es <ol>?
Es una lista ordenada, normalmente numerada.
¿Qué es <li>?
Es un ítem individual dentro de una lista.
¿Qué es <div>?
Es un contenedor genérico de bloque, muy usado para agrupar contenido y estructurarlo con CSS o JavaScript.
¿Qué es <span>?
Es un contenedor genérico en línea, útil para aplicar estilos o comportamiento a una parte pequeña del texto.
¿Qué es una etiqueta semántica?
Es una etiqueta que expresa el significado del contenido que contiene, como <header>, <nav>, <main> o <footer>.
¿Qué es <header>?
Representa el encabezado de una página o de una sección.
¿Qué es <nav>?
Representa una sección de navegación, como menús o enlaces principales.
¿Qué es <main>?
Representa el contenido principal del documento.
¿Qué es <section>?
Representa una sección temática del documento.
¿Qué es <article>?
Representa un contenido independiente y autocontenido, como una noticia, publicación o post.
¿Qué es <aside>?
Representa contenido secundario o complementario al contenido principal, como una barra lateral.
¿Qué es <footer>?
Representa el pie de página o pie de una sección.
¿Qué es un formulario?
Es una sección que permite al usuario ingresar y enviar datos a un servidor.
¿Qué es <form>?
Es el contenedor principal de un formulario HTML. Agrupa campos y define cómo se enviarán los datos.
¿Qué hacen action y method en un <form>?
action indica a qué URL se enviarán los datos. method indica el método HTTP usado, normalmente GET o POST.
¿Qué es <input>?
Es un campo de entrada que permite capturar datos del usuario. Su comportamiento cambia según el atributo type.
¿Qué es <label>?
Es una etiqueta descriptiva asociada a un campo de formulario. Mejora accesibilidad y usabilidad.
¿Para qué sirve el atributo for en <label>?
Sirve para vincular la etiqueta <label> con un campo específico mediante su id.
¿Qué es <textarea>?
Es un campo de formulario pensado para ingresar texto largo.
¿Qué es <select>?
Es una lista desplegable que permite elegir una opción entre varias.
¿Qué es <option>?
Es una opción individual dentro de un <select>.
¿Qué es <button>?
Es un botón interactivo que puede enviar formularios o ejecutar acciones.
¿Qué es <table>?
Es la etiqueta que define una tabla HTML.
¿Qué es <tr>?
Representa una fila dentro de una tabla.
¿Qué es <td>?
Representa una celda de datos dentro de una fila.
¿Qué es <th>?
Representa una celda de encabezado dentro de una tabla.
3. CSS
¿Qué es CSS?
Es el lenguaje que define la presentación visual de un documento HTML: colores, tamaños, márgenes, posiciones y diseño.
¿Para qué sirve CSS?
Sirve para separar la estructura del contenido de su presentación visual, haciendo el código más limpio, reutilizable y mantenible.
¿Qué es una regla CSS?
Es una instrucción formada por un selector y un bloque de declaraciones que indica cómo se deben mostrar ciertos elementos HTML.
¿Qué es un selector CSS?
Es la parte de la regla que indica a qué elemento o elementos se les aplicará el estilo.
¿Qué es una propiedad CSS?
Es la característica visual que se quiere modificar, por ejemplo color, margin o font-size.
¿Qué es un valor en CSS?
Es el dato asignado a una propiedad, por ejemplo blue, 16px o center.
Tipos de selectores CSS
Los más comunes son: selector de elemento, de clase, de ID, de atributo, universal, pseudo-clase, pseudo-elemento, descendiente, hijos directos y hermanos.
¿Qué es un selector de clase?
Selecciona todos los elementos que comparten el mismo valor en el atributo class. Se escribe con punto, por ejemplo .destacado.
¿Qué es un selector de ID?
Selecciona un elemento específico por su atributo id. Se escribe con numeral, por ejemplo #encabezado.
¿Qué es un selector descendiente?
Selecciona elementos que están dentro de otro elemento, aunque no sean hijos directos. Ejemplo: div a.
¿Dónde se puede escribir CSS?
Se puede escribir de tres formas: en un archivo externo .css, dentro de una etiqueta <style> en el <head>, o en línea con el atributo style.
¿Cómo se vincula una hoja de estilos externa?
Se usa la etiqueta <link> dentro de <head>, por ejemplo: <link href="styles/style.css" rel="stylesheet">.
¿Qué es CSS en línea?
Es escribir estilos directamente en el atributo style de una etiqueta HTML.
¿Qué es CSS interno?
Es escribir reglas CSS dentro de una etiqueta <style> en el documento HTML.
¿Qué es CSS externo?
Es escribir los estilos en un archivo separado con extensión .css y vincularlo al HTML.
¿Qué es la especificidad en CSS?
Es la regla que determina qué estilo gana cuando varias reglas compiten sobre el mismo elemento. Un selector más específico prevalece sobre uno menos específico.
¿Qué pasa si dos reglas tienen la misma especificidad?
Gana la que aparece más tarde en la cascada.
¿Qué prioridad tiene el estilo en línea?
Tiene mayor prioridad que las reglas internas o externas normales.
¿Qué son las variables CSS?
Son propiedades personalizadas que permiten guardar valores reutilizables, como colores o tamaños, y referenciarlos luego con var(...).
¿Qué hace !important?
Fuerza a que una regla tenga prioridad sobre otras, incluso si normalmente perdería por cascada o especificidad.
4. JavaScript y Node.js
¿Qué es JavaScript?
Es un lenguaje de programación usado para agregar lógica e interactividad a las aplicaciones web. Hoy también puede ejecutarse fuera del navegador.
¿JavaScript y Java son lo mismo?
No. Son lenguajes diferentes, con historia, diseño y modelo de ejecución distintos.
¿Qué es ECMAScript?
Es la especificación estandarizada de JavaScript, que define cómo debe comportarse el lenguaje.
¿Qué es Node.js?
Es un entorno de ejecución que permite correr JavaScript fuera del navegador, especialmente del lado del servidor.
¿Qué es un runtime?
Es el entorno donde corre un lenguaje, incluyendo motor de ejecución, APIs y recursos del sistema.
¿Qué es el motor V8?
Es el motor de JavaScript de Google que usan Chrome y Node.js para ejecutar código JavaScript eficientemente.
¿Qué significa Single Thread?
Significa que el hilo principal de ejecución procesa una instrucción a la vez. Node.js ejecuta JavaScript en un único hilo principal.
¿Qué es un thread?
Es una unidad de ejecución dentro de un proceso.
¿Qué es un proceso?
Es una instancia de un programa en ejecución, con memoria y recursos propios.
¿Qué es el Event Loop?
Es el mecanismo que revisa constantemente si hay tareas pendientes y, cuando el hilo principal queda libre, ejecuta la siguiente disponible.
¿Qué es asincronía?
Es la capacidad de iniciar operaciones lentas sin bloquear el hilo principal, permitiendo que el programa siga haciendo otras tareas.
Bloqueante vs no bloqueante
En una operación bloqueante, el programa espera hasta terminarla. En una no bloqueante, inicia la tarea y continúa trabajando hasta recibir el resultado.
¿Qué es un callback?
Es una función que se pasa como parámetro para ejecutarse más adelante, cuando termina una tarea o sucede un evento.
¿Qué es una Promise?
Es un objeto que representa el resultado futuro de una operación asíncrona. Puede estar pendiente, resuelta o rechazada.
¿Qué es async / await?
Es una sintaxis que permite trabajar con promesas de forma más legible, escribiendo asincronía con apariencia secuencial.
¿Qué es el Call Stack?
Es la estructura donde se apilan las funciones activas. Cuando una función termina, se desapila.
¿Qué es una API REST?
Es una interfaz donde cliente y servidor se comunican mediante recursos y métodos HTTP como GET, POST, PUT y DELETE.
¿Qué hace GET?
Solicita datos al servidor.
¿Qué hace POST?
Envía datos al servidor para crear o registrar nueva información.
¿Qué hace PUT?
Reemplaza o actualiza un recurso existente.
¿Qué hace DELETE?
Elimina un recurso del servidor.
¿Cómo se vincula JavaScript al HTML?
Se puede escribir dentro de una etiqueta <script> o vincular un archivo externo con <script src="archivo.js"></script>.
5. Programación
¿Qué es un algoritmo?
Es una secuencia ordenada y finita de pasos para resolver un problema.
¿Qué es una variable?
Es un espacio de memoria con nombre donde se almacena un valor.
¿Qué es un tipo de dato?
Es la categoría del valor que determina qué información contiene y qué operaciones se pueden hacer sobre él.
¿Qué es una condición?
Es una expresión que se evalúa como verdadera o falsa y permite tomar decisiones en el programa.
¿Qué es un bucle?
Es una estructura que repite instrucciones mientras se cumpla una condición o una cantidad determinada de veces.
¿Qué es una función?
Es un bloque de código reutilizable que realiza una tarea específica. Puede recibir datos y devolver un resultado.
¿Qué es un parámetro?
Es la variable declarada en la definición de una función para representar los datos que espera recibir.
¿Qué es un argumento?
Es el valor real que se le pasa a la función cuando se la invoca.
Diferencia entre parámetro y argumento
El parámetro aparece en la definición de la función. El argumento es el valor concreto enviado cuando la función se ejecuta.
¿Qué significa retornar?
Significa devolver un valor como resultado de una función.
¿Qué es el scope o alcance de una variable?
Es la parte del programa donde esa variable existe y puede ser usada.
¿Qué es un array?
Es una estructura que guarda varios valores en una misma colección, ordenados por posición o índice.
¿Qué es un objeto?
Es una estructura que agrupa información relacionada, normalmente en pares clave-valor, y según el paradigma también comportamientos.
¿Qué es modularidad?
Es la división del programa en partes independientes y organizadas, cada una con una responsabilidad clara.
¿Qué es reutilización de código?
Es la posibilidad de usar una misma solución o bloque de código en varios lugares sin duplicarlo.
¿Qué es programación funcional?
Es un paradigma que organiza la solución alrededor de funciones y transformación de datos, buscando minimizar cambios de estado innecesarios.
¿Qué es programación orientada a objetos?
Es un paradigma que modela el sistema con objetos que combinan estado y comportamiento.
Diferencia entre funcional y objetos
La programación funcional pone el foco en funciones y transformación de datos. La orientada a objetos pone el foco en entidades con atributos y comportamientos que colaboran entre sí.
¿Qué paradigma usa JavaScript?
JavaScript es multiparadigma. Puede programarse de forma funcional, orientada a objetos y también imperativa.
